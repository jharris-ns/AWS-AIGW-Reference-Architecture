# DLPoD Automation

Automated provisioning, tethering, and AI Gateway integration of Netskope DLP On Demand appliances via Lambda + Step Functions.

## Architecture

The DLPoD appliance uses a **traditional CLI shell** (`nsappliance>` prompt), not a TUI like the AI Gateway. Automation uses the same paramiko + pyte infrastructure with `ParamikoTUISession` in `mode='cli'`.

DLPoD runs in its own Auto Scaling Group with lifecycle hooks, mirroring the AIG enrollment pattern. All DLPoD resources are conditional on `DeployDlpod` in `gateway-asg.yaml`.

```
CloudFormation deploys:
  - DLPoD ASG with lifecycle hooks
  - Private ALB (HTTPS, self-signed CA cert)
  - Route 53 private hosted zone (dlp.aigw.internal → ALB)
  - DLPoD Activation Lambda + Step Functions for tethering
  - Self-signed cert via custom resource → ACM import + SSM storage

CloudFormation deploys DLPoD ASG with lifecycle hooks
        |
ASG launches DLPoD instance → lifecycle hook holds in Pending:Wait
        |
SNS → DLPoD Activation Lambda (generates password, starts SFN)
        |
Step Functions tethering state machine
   SSH to DLPoD as nsadmin with generated password
   Drives nsappliance CLI for each step
        |
CompleteLifecycleAction → instance moves to InService

AIG DLP configuration (per gateway instance, after enrollment):
  Read cert from SSM → SSH to AIG TUI → bracket-paste cert → set host URL
```

## DLPoD CLI Reference

The DLPoD runs Netskope Appliance firmware (tested: v135.0.25). The CLI shell is entered immediately on SSH login as `nsadmin`.

### Prompts
- Normal mode: `nsappliance>`
- Configure mode: `nsappliance(config)#`

### Key Commands

| Command | Mode | Purpose |
|---------|------|---------|
| `configure` | normal | Enter configure mode |
| `save` | config | Persist configuration |
| `exit` | config | Return to normal mode |
| `auth change-password nsadmin` | normal | Change nsadmin password (prompts: New password, Retype new password) |
| `set dns primary <ip>` | config | Set primary DNS server |
| `set dns secondary <ip>` | config | Set secondary DNS server |
| `set system licensekey <key>` | config | Set license key for tenant tethering |
| `show dns` | normal | Display DNS configuration |
| `status tethering` | normal | Show tethering status (JSON output) |
| `status all` | normal | Show all component status |
| `restart dlpaas all` | normal | Restart all DLP services |
| `run request certificate generate self-signed ...` | config | Generate self-signed TLS cert |
| `request system reboot` | normal | Reboot appliance |

## DLPoD Tethering Flow

### Step Functions State Machine

```
MergePassword (Pass state — merges generated password into execution state)
    |
WaitForDlpodSSH (poll every 15s)
    |
ChangePassword (auth change-password nsadmin)
    |
SetDNS (set dns primary <vpc-resolver>)
    |
SetLicenseKey (configure + set system licensekey + save)
    |
WaitForTetheringInit (120s)
    |
CheckTethering (poll every 60s until tethered)
    |
DlpodCompleteLifecycle (CompleteLifecycleAction → InService)
```

Passwords are generated by the activation Lambda and passed through the Step Functions execution state (not Secrets Manager). This avoids race conditions when multiple DLPoD instances launch simultaneously — each execution carries its own password.

### Timing (observed, 2026-05-11)

| Phase | Duration |
|-------|----------|
| Instance boot to SSH ready | ~5 min |
| Password change + DNS + license | ~30s |
| Tethering (after license set + DNS fix) | ~5 min |
| **Total** | **~10 min** |

## ALB and Certificate Architecture

The DLPoD runs behind a **private internal ALB** to provide a stable DNS endpoint with a trusted certificate.

```
AI Gateway → https://dlp.aigw.internal → ALB (self-signed CA cert) → DLPoD :443
```

### Certificate Requirements

The AIG validates the DLP service certificate with these requirements:
1. **Must be a CA certificate** — `basicConstraints = critical, CA:TRUE`
2. **CN or SAN must match the DLP host URL** hostname
3. **PEM format** — BEGIN/END markers on separate lines, base64 body on a **single line** (no line breaks within the base64 content)

The cert is generated by a CloudFormation custom resource (Lambda + openssl), imported into ACM for the ALB HTTPS listener, and stored in SSM Parameter Store for the AIG TUI automation to read.

### Route 53

A private hosted zone (`aigw.internal`) is created in the VPC with an alias record (`dlp.aigw.internal`) pointing to the ALB. All instances in the VPC can resolve this name.

## AIG DLP Configuration

After the AI Gateway is enrolled, the DLP service is configured via the AIG TUI.

### TUI Navigation

```
Main Menu → Configure Content Inspection Services
  → Data Loss Prevention Service
    → Configure DLP Service Certificate  (paste cert PEM)
    → Configure DLP Service Host         (enter https://dlp.aigw.internal)
```

### Bracket Paste Mode

The AIG TUI uses a Bubble Tea text input widget that requires **bracketed paste mode** for multi-line content. Raw `send()` or line-by-line input does not work — the text is not received by the widget.

```python
PASTE_START = '\x1b[200~'
PASTE_END = '\x1b[201~'
session.child.send(PASTE_START + cert_content + PASTE_END)
```

### Certificate Format for TUI Paste

The cert must be collapsed into 3 lines before pasting:
```
-----BEGIN CERTIFICATE-----
<entire base64 on ONE line — no line breaks>
-----END CERTIFICATE-----
```

Standard PEM files have 64-char line wrapping. The AIG TUI rejects certs with line breaks in the base64 body.

## Findings and Gotchas

### DLPoD CLI
1. **Password command requires username**: `auth change-password nsadmin`, not `auth change-password`
2. **No current password prompt**: Goes directly to `New password:` then `Retype new password:`
3. **Password complexity**: Alphanumeric-only passwords rejected as "BAD PASSWORD: is too simple". Must include uppercase, lowercase, digits, and special characters.
4. **DNS not provided by DHCP**: The DLPoD cannot resolve Netskope management plane hostnames without explicit DNS configuration. For AWS VPCs, set DNS to VPC CIDR base + 2 (e.g., `172.31.0.2` for `172.31.0.0/16`).
5. **DNS must be set before license key**: Without DNS, the appliance accepts the license key but cannot reach the management plane for tethering.
6. **Service restart after DNS change**: `restart dlpaas all` is needed for DNS changes to take effect on tethering services.
7. **c5ad not available everywhere**: c5ad instances have limited AZ availability. Use c5a as fallback (same compute, no local NVMe).

### Certificate and TLS
8. **CA:TRUE required**: The AIG's `aig-dp-mgmt-svc` rejects certs with `CA:FALSE` — error: "certificate must be a CA certificate".
9. **Single-line base64**: The AIG TUI rejects certs with line breaks in the base64 body — error: "no certificate block found".
10. **Bracket paste mode**: The AIG TUI (Bubble Tea framework) requires `\x1b[200~`..`\x1b[201~` escape sequences to paste multi-line content into text input widgets. Raw `send()` or `\r`-delimited input is not received.
11. **Reboot after cert generation**: The DLPoD HTTPS daemon does not pick up a new certificate until the appliance reboots. This is not documented by Netskope.

### Infrastructure
12. **ALB needs large subnets**: ALB requires at least 8 free IPs per subnet. The small `/28` private subnets don't have enough — use `/20` or larger subnets for the ALB.
13. **Cert storage**: The DLPoD server certificate is a public cert (not a secret). SSM Parameter Store (`String` type) is appropriate. Secrets Manager would be unnecessary cost.
14. **Passwords passed through SFN state**: Each DLPoD instance gets a unique password generated by the activation Lambda and passed through the state machine execution state. This avoids race conditions when multiple instances launch simultaneously.
15. **Log redaction**: Sensitive fields (password, license_key, enrollment_token, api_token) are redacted with `***` in Lambda CloudWatch logs.

## Deployment

Deploy via the main `gateway-asg.yaml` template with `DeployDlpod=true`. The template exceeds 51KB when DLPoD resources are included, so use `--template-url` with an S3-hosted template.

```bash
# Build and upload Lambda packages + template to S3
scripts/deploy-artifacts.sh us-west-1

# Deploy with DLPoD enabled
aws cloudformation create-stack \
  --stack-name <name> \
  --template-url https://<s3-bucket>.s3.us-west-1.amazonaws.com/templates/gateway-asg.yaml \
  --parameters \
    ParameterKey=NetskopeTenantUrl,ParameterValue=https://tenant.goskope.com \
    ParameterKey=NetskopeApiToken,ParameterValue=<token> \
    ParameterKey=GatewayAmiId,ParameterValue=<ami-id> \
    ParameterKey=LambdaCodeBucket,ParameterValue=<s3-bucket> \
    ParameterKey=DlpodAmiId,ParameterValue=<dlpod-ami> \
    ParameterKey=DlpodLicenseKey,ParameterValue=<license-key> \
  --capabilities CAPABILITY_NAMED_IAM \
  --region us-west-1
```

For standalone testing, the test template is still available:

```bash
# Deploy test stack (DLPoD + ALB + cert + automation only)
aws cloudformation create-stack \
  --stack-name dlpod-alb-test \
  --template-body file://templates/test-dlpod.yaml \
  --parameters \
    ParameterKey=VpcId,ParameterValue=<vpc-id> \
    'ParameterKey=AlbSubnetIds,ParameterValue=<subnet-az1>,<subnet-az2>' \
    'ParameterKey=PrivateSubnetIds,ParameterValue=<private-subnet-az1>,<private-subnet-az2>' \
    ParameterKey=LambdaCodeBucket,ParameterValue=<s3-bucket> \
    ParameterKey=DlpodAmiId,ParameterValue=<dlpod-ami> \
    ParameterKey=DlpodLicenseKey,ParameterValue=<license-key> \
    ParameterKey=DnsServer,ParameterValue=<vpc-dns-ip> \
  --capabilities CAPABILITY_NAMED_IAM \
  --region us-west-1
```
